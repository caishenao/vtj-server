# VTJ 低代码管理平台需求文档 (PRD)

## 1. 文档概览
- **项目名称**：VTJ Low-Code Management Platform
- **目标**：提供一个统一的低代码应用管理门户，支持应用的创建、可视化设计、权限控制及菜单挂载。

## 2. 角色定义
- **系统管理员**：管理平台基础配置、多租户管理、日志监控。
- **应用开发者**：创建低代码应用、使用 VTJ 设计器编写 DSL、发布应用。
- **最终用户**：在管理后台通过菜单访问并使用具体的低代码应用。

## 3. 功能需求
### 3.1 应用管理模块
- **应用 CRUD**：支持创建应用名称、唯一标识（AppKey）、描述。
- **状态管理**：应用可处于“开发中”、“已发布”、“已下线”状态。
- **应用列表**：展示所有已创建的低代码应用，支持按租户/部门过滤。

### 3.2 可视化设计集成
- **设计器嵌入**：在 Vben 中通过 Iframe 或直接组件嵌入 VTJ 设计器。
- **DSL 持久化**：设计器保存时，将生成的 JSON DSL 发送到后端存储到 PostgreSQL。
- **物料管理**：支持上传自定义物料组件，并在设计器中动态加载。

### 3.3 菜单与权限绑定
- **菜单挂载**：在“菜单管理”中，可以将某个菜单项类型标记为“低代码应用”，并选择已有的应用。
- **权限继承**：VTJ 应用作为菜单项，自动继承 RuoYi 的 RBAC 权限控制，只有被分配了该菜单权限的用户才能看到并进入应用。

### 3.4 动态路由获取
- **接口定义**：提供 /api/low-code/routes 接口，供前端渲染器获取当前用户有权访问的所有低代码应用路由。
- **路由转换**：后端根据 sys_menu 中的 pp_id 自动转换为 Vben 识别的 RouteRecord 格式。

## 4. 数据模型设计 (核心表)
### 4.1 sys_lc_app (低代码应用表)
- id: 主键
- pp_name: 应用名称
- pp_key: 唯一标识
- dsl_json: VTJ DSL 数据 (JSONB)
- config: 应用全局配置 (JSONB)
- 	enant_id: 租户 ID

### 4.2 sys_menu (系统菜单表扩展)
- pp_id: 关联的低代码应用 ID (可选)

## 5. 交互流程
1. **创建应用**：开发者在“应用管理”中新建应用。
2. **设计页面**：点击“设计”，进入 VTJ 设计器，编辑完成后保存。
3. **挂载菜单**：在“菜单管理”中新建菜单，组件指向 VTJContainer，参数填入 pp_id。
4. **用户访问**：最终用户登录系统，点击对应菜单，系统自动加载并渲染低代码页面。

## 6. 非功能需求
- **性能**：VTJ 渲染引擎应按需加载（Code Splitting），避免影响首屏速度。
- **兼容性**：支持主流浏览器，适配 Vben5 的暗黑模式。
